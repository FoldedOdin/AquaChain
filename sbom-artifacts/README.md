# SBOM Artifacts Directory

This directory contains generated Software Bill of Materials (SBOMs) and vulnerability reports for the AquaChain project.

## Directory Contents

### SBOM Files

- `sbom-frontend-<timestamp>.json` - Frontend (React) dependencies
- `sbom-backend-<timestamp>.json` - Backend (Lambda) dependencies
- `sbom-infrastructure-<timestamp>.json` - Infrastructure (CDK) dependencies
- `sbom-iot-firmware-<timestamp>.json` - IoT firmware dependencies
- `sbom-complete-<timestamp>.json` - Merged comprehensive SBOM

### Vulnerability Reports

- `sbom-<component>-<timestamp>.vulnerabilities.json` - Raw Grype scan results
- `vulnerability-report-<component>-<timestamp>.json` - Structured vulnerability data
- `vulnerability-report-<component>-<timestamp>.html` - Human-readable HTML report

### Summary Reports

- `sbom-report-<timestamp>.json` - Summary of all SBOMs and vulnerabilities
- `sbom-comparison-<timestamp>.json` - Comparison between SBOM versions

### Checksums

- `*.sha256` - SHA256 checksums for integrity verification

## File Formats

### SBOM Format (SPDX 2.3 JSON)

```json
{
  "spdxVersion": "SPDX-2.3",
  "dataLicense": "CC0-1.0",
  "SPDXID": "SPDXRef-DOCUMENT",
  "name": "AquaChain-Complete-SBOM",
  "packages": [...],
  "relationships": [...]
}
```

### Vulnerability Report Format

```json
{
  "component": "frontend",
  "timestamp": "2025-10-25T10:00:00Z",
  "summary": {
    "Critical": 0,
    "High": 2,
    "Medium": 5,
    "Low": 12
  },
  "total_vulnerabilities": 19,
  "vulnerabilities_by_severity": {...}
}
```

## Usage

### View HTML Report

Open any `.html` file in a web browser:

```bash
open vulnerability-report-frontend-*.html
```

### Parse JSON Reports

```bash
# Count critical vulnerabilities
jq '.summary.Critical' sbom-report-*.json

# List all vulnerable packages
jq '.vulnerabilities_by_severity.Critical[].package' vulnerability-report-*.json

# Get total package count
jq '.total_packages' sbom-report-*.json
```

### Verify Checksums

```bash
# Verify a specific file
sha256sum -c sbom-complete-*.json.sha256

# Verify all files
for file in *.sha256; do
  sha256sum -c "$file"
done
```

## Retention

Local artifacts are retained based on your local storage policies. For long-term retention:

- **S3 Storage:** All SBOMs are uploaded to `s3://aquachain-sbom-<account-id>/`
- **Retention Period:** 
  - Latest: Indefinite
  - Weekly: 1 year
  - Timestamped: 90 days (non-current versions)

## Accessing Historical SBOMs

```bash
# List all SBOMs in S3
aws s3 ls s3://aquachain-sbom-<account-id>/

# Download specific SBOM
aws s3 cp s3://aquachain-sbom-<account-id>/latest/sbom-complete-*.json ./

# Download entire week
aws s3 sync s3://aquachain-sbom-<account-id>/weekly/2025-W43/ ./
```

## Cleanup

To clean up old local artifacts:

```bash
# Remove artifacts older than 30 days
find sbom-artifacts/ -name "*.json" -mtime +30 -delete
find sbom-artifacts/ -name "*.html" -mtime +30 -delete
find sbom-artifacts/ -name "*.sha256" -mtime +30 -delete
```
